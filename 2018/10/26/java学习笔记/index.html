常用数学函数

三角函数

  • sin(radians)–返回正弦值
  • cos(radians)
  • tan(radians)
  • toRadians(degree):以度为单位的角度值转换为以弧度表示
  • toDegrees(radians):以弧度为单位的角度值转换为以度表示
  • asin(a):返回以弧度为单位的角度的反三角正弦函数值。范围：-π/2~π/2
  • acos(a):返回**。范围：0~π
  • atan(a):返回****.范围：-π/2~π/2

指数函数

  • exp(x)：返回e的x次方
  • log(x)：返回x的自然底数
  • log10(x)：返回x以10为底的对数
  • pow(a,b)：返回a的b次方
  • sqrt(x)：对于x>=0的数字，返回x的平方根
Math.exp(1)//2.71828
Math.log(Math.E)//1.0
Math.log10(10)//1.0
Math.pow(2,3)//8.0
Math.pow(3,2)//9.0
Math.sqrt(4)//2.0

取整函数

  • ceil(x)：向上取整，作为双精度返回
  • floor(x)：向下取整，作为双精度返回
  • rint(x)：取整为最接近的数。如果x与两个证书的距离相等，偶数的整数作为一个双精度值返回
  • round(x)：如果x是单精度，返回(int)Math.floor(x+0.5)；如果x是双精度，返回(long)Math.floor(x+0.5)
Math.ceil(2.1)//3.0
Math.ceil(-2.1)//-2.0
Math.floor(2.1)//2
Math.floor(-2.1)//-3
Math.rint(2.1)//2
Math.rint(2.5)//2
Math.rint(4.5)//4
Math.round(2.6f)//3(int)
Math.round(2.0)//2(long int)
Math.round(-2.6)//-3
Math.round(-2.4)//-2

max,min,abs

max(a,b),min(a,b)返回大的和小的值。
abs(x)，返回绝对值。

    继承

关键字：extends

特性

  • 子类拥有父类非private的属性和方法
  • 子类可以拥有自己的属性和方法，即子类可以对父类进行扩展。
  • 子类可以用自己的方式实现父类的方法。
  • Java的继承是单继承，但是可以多重继承，单继承就是一个子类只能继承一个父类，多重继承就是，例如A类继承B类，B类继承C类，所以按照关系就是C类是B类的父类，B类是A类的父类，这是java继承区别于C++继承的一个特性。
  • 提高了类之间的耦合性（继承的缺点，耦合度高就会造成代码之间的联系越紧密，代码独立性越差）。

关键字

extends

在 Java 中，类的继承是单一继承，也就是说，一个子类只能拥有一个父类，所以 extends 只能继承一个类。

public class Animal {
    private String name;

        //初始化属性值
    }
    public void eat() {  //吃东西方法的具体实现  }
    public void sleep() { //睡觉方法的具体实现  }
}

public class Penguin  extends  Animal{
}

implements

使用 implements 关键字可以变相的使java具有多继承的特性，使用范围为类继承接口的情况，可以同时继承多个接口（接口跟接口之间采用逗号分隔）。

public interface A {
    public void eat();
    public void sleep();
}

public interface B {
    public void show();
}

public class C implements A,B {
}

super and this

  • super关键字：我们可以通过super关键字来实现对父类成员的访问，用来引用当前对象的父类。

  • this关键字：指向自己的引用。

eg

class Animal {
  void eat() {
    System.out.println("animal : eat");
  }
}

class Dog extends Animal {
  void eat() {
    System.out.println("dog : eat");
  }
  void eatTest() {
    this.eat();   // this 调用自己的方法
    super.eat();  // super 调用父类方法
  }
}

public class Test {
  public static void main(String[] args) {
    Animal a = new Animal();
    a.eat();
    Dog d = new Dog();
    d.eatTest();
  }
}
---------------------
输出结果为：

animal : eat
dog : eat
animal : eat

重写(Override) and 重载(Overload)

Override

重写是子类对父类的允许访问的方法的实现过程的重新编写，返回值和形参不能改变，也就是外壳不变，核心重写。
举个例子 有如下代码

class animal {
    public void act() {
        System.out.println("All of the animals can run!");
    }

}

class horse extends animal{
    public void move() {
        System.out.println("The horse can run fastly!!");
    }
    public void bark() {
        System.out.println("The horse can bark!");
    }
}

public class override {
    public static void main(String[] args) {
        animal a = new animal();
        animal b = new horse();

        a.act();
        b.act();
        b.bark();

    }

}

如果在eclipse中跑这段程序是会报错 报错为没有为类型 animal 定义方法 bark()，也就是说b引用的animal类型没有bark方法。

那么就要说到重写规则

  • 参数列表必须完全与被重写方法的相同；
  • 返回类型必须完全与被重写方法的返回类型相同；
  • 访问权限不能比父类中被重写的方法的访问权限更低。例如：如果父类的一个方法被声明为public，那么在子类中重写该方法就不能声明为protected。
  • 父类的成员方法只能被它的子类重写。
  • 声明为final的方法不能被重写。
  • 声明为static的方法不能被重写，但是能够被再次声明。
  • 子类和父类在同一个包中，那么子类可以重写父类所有方法，除了声明为private和final的方法。
  • 子类和父类不在同一个包中，那么子类只能够重写父类的声明为public和protected的非final方法。
  • 重写的方法能够抛出任何非强制异常，无论被重写的方法是否抛出异常。但是，重写的方法不能抛出新的强制性异常，或者比被重写方法声明的更广泛的强制性异常，反之则可以。
  • 构造方法不能被重写。
  • 如果不能继承一个方法，则不能重写这个方法。

需要在子类中调用父类的被重写方法时，就要用到super关键字

class animal {
    public void move() {
        System.out.println("All of the animals can run!");
    }

}

class horse extends animal{
    public void move() {
        super.move();
        System.out.println("The horse can run fastly!!");
    }

}

public class override {
    public static void main(String[] args) {
        animal a = new horse();

        a.move();

    }
}
------------
result:
All of the animals can run!
The horse can run fastly!!

Overload

重载(Overload)是在一个类里面，方法名字相同，而参数不同，返回类型可以不同。
每个重载的方法（构造函数）都必须有一个独一无二的参数类型列表。比较常用的就是构造器的重载。

重载规则

  • 被重载的方法必须改变参数列表(参数个数或类型不一样)；
  • 被重载的方法可以改变返回类型；
  • 被重载的方法可以改变访问修饰符；
  • 被重载的方法可以声明新的或更广的检查异常；
  • 方法能够在同一个类中或者在一个子类中被重载。
  • 无法以返回值类型作为重载函数的区分标准。

code

public class Overload {
    public int strone() {
        System.out.println("This is the first number!");
        return 1;
    }
    public void strone(int a) {
        System.out.println("This is the second number!");
    }
    //下面两个参数类型顺序是相反的
    public String txt(int a, String s) {
        System.out.println("This is the  third number!");
        return "gugugu";
    }
    public String txt(String s,int a) {
        System.out.println("This is the forth numebr!");
        return "ying ying ying";
    }
    public static void main(String[] args) {
        Overload str = new Overload();
        System.out.println(str.strone());
        str.strone(1);
        System.out.println(str.txt(1,"asdaweqea"));
        System.out.println(str.txt("asdasdasdu",123));
    }

}
----------------
result:
This is the first number!
1
This is the second number!
This is the  third number!
gugugu
This is the forth numebr!
ying ying ying

总而言之，Override和Overload是java多态性的不同表现，重写是父类和子类之间多态性的一种表现，重载可以理解为多台的具体表现形式。

override：方法名和参数都一样 overload：方法名相同，但是参数不同

如果在定义一个类的时候没有指定继承性，那么久默认为这个类的父类是Object。java中很多类都是Object的隐含子类。

多态

多态意味着父类的变量可以指向子类对象
当使用多态方式调用方法时，首先检查父类中是否有该方法，有的话才能去调用子类的同名方法。多态的好处就是可以使程序有良好的扩展，可以对所有类的对象进行通用处理。
还要了解两个术语：子类型(subtype)和父类型(supertype)，也就是说一个类实际上定义了一种类型.

动态绑定

原理：假设有一个对象O是类c1,c2,c3,c4…..cn的实例，其实c1是c2的子类，以此类推，即cn是最通用的类，c1是最特殊的类。然后cn是object类。如果O调用一个方法，那么JVM会在类c1,c2,c3….直到cn中寻找这个方法的实现，直到找到为止

Cn <- Cn-1 <-......<-C2 <- C1
Cn为 java.lang.Object

举个例子
这里我们创建一个父类Father，然后一个子类Son继承Father类

public class Son extends Father {
    @Override
    public void say() {
        System.out.println("儿子说话");
    }

    @Override
    public void sing() {
        System.out.println("儿子唱歌");
    }

    @Override
    public void dance() {
        System.out.println("儿子跳舞");
    }

    public static void main(String[] args) {
        Father father = new Son();
        father.say();
        father.run();
    }
}

class Father{
    public void say() {
        System.out.println("爸爸说话");
    }

    public void sing() {
        System.out.println("爸爸唱歌");
    }

    public void dance() {
        System.out.println("爸爸跳舞");
    }

    public void run() {
        System.out.println("爸爸跑步");
    }
}
---------------------------
result:
儿子说话
爸爸跑步

对象转换和instanceof运算符

前言：对象的引用可以类型转换为对另外一种对象的引用，这叫对象转换。
比如m(new student());是将对象new student()赋值给一个Object类型的参数，等价于

Object o = new student();
m(o)

由于student的实例也是Object的实例，所以上面的语句是合法的，这叫做隐式转换。

但是可以Object o = new student();，但是却不可以student b = o，因为student的对象总是Object的实例，但是Object的实例不一定是student的实例。
举个例子：这里有fruit,apple,orange。其中fruit是另外两个的父类，apple是fruit，那么apple的所有实例都可以赋给fruit，但是不一定所有fruit都是apple，所以这里就要进行转换一下才能赋值。
所以要改成student b = (student)o;这叫做向上转换，子类的实例永远是父类的实例。那么在转换之前保证两个对象的实例一样，那就不会运行异常（ClassCastException），这就用到了instanceof来实现，注：java中所有的关键字都是小写的

ArrayList类

作用

用于存储一个对象列表！传说中的动态数组，这里给个比较简单的例子助于理解
创建一个数字列表，然后输入不同的数字，输入0程序结束！

import java.util.ArrayList;
import java.util.Scanner;

public class Demo {
    public static void main(String[] args) {
        ArrayList<Integer> xulie = new ArrayList<>();
        //创建一个列表，用来存储输入的数字

        Scanner input = new Scanner(System.in);
        System.out.print("Please input some numbers,if you input zero ,game is over! ");
        int number;

        do {
            number = input.nextInt();

        if(!xulie.contains(number) && number !=0)
        {
            xulie.add(number);
        }
        }
        while(number != 0);
        for(int i = 0; i<xulie.size();i++)
        {
            System.out.println(xulie.get(i) + " ");
        }

    }
}

其实，动态数组给我的感觉就像是栈一样。
